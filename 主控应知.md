# 2023 E 题视觉协议与主控实现说明

本文以当前代码和官方赛题为准。最重要的分工是：

```text
MaixCAM：识别边框/光斑、生成目标、计算像素误差、判断到达、切换航点
STM32：解析串口、按 DX/DY 做双轴 PID、驱动云台、制动、按键、声光提示
```

主控不要把像素坐标直接换算成电机步数。`TYPE=0x01` 是任务状态，`TYPE=0x02` 才是 PID 输入。

## 1. 先选红色任务模式，再执行

| 模式号 | 名称 | 视觉目标来源 | 航点数 |
|---:|---|---|---:|
| `0` | 原点复位 | 检测屏幕上 0.5 m 正方形，取四角平均中心 | 1 |
| `1` | 屏幕正方形巡边 | 检测屏幕上 0.5 m 正方形，四边各五等分 | 20 |
| `2` | A4 靶纸巡边 | 检测 A4 黑胶带中心线或退化单矩形 | 20 |

题面“原点”是屏幕正方形中心，不是 A4 中心，也不能未经标定就假定是图像 `(320,240)`。

红色程序上电默认模式 `2`。主控或摄像头顶部触摸按钮选择模式后，视觉会清除旧轨迹并重新识别。主控可以紧接着发送开始命令；如果仍在识别，视觉会记录“待启动”，锁定后自动运行，不会丢掉这次开始操作。

摄像头屏幕布局：

```text
顶部：[ORIGIN] [SCREEN] [A4]       当前模式绿色高亮
底部：[REACQUIRE] [START/RESUME] [PAUSE]
```

原点和屏幕正方形模式锁定后，屏幕会用白色圆圈和十字绘制原点。A4 模式不会把 A4 中心当作题面原点。

## 2. 串口参数与固定帧

| 项目 | 当前约定 |
|---|---|
| 波特率 | `115200` |
| MaixCAM 串口 | `/dev/ttyS1` |
| MaixCAM TX/RX | `A19 / A18`，与主控交叉连接并共地 |
| 帧长 | 固定 9 字节 |
| 16 位字节序 | 小端 |
| 校验 | 前 8 字节累加和低 8 位 |

```text
Byte:      0    1    2      3      4       5       6       7       8
           AA   55   TYPE   INDEX  DATA0L  DATA0H  DATA1L  DATA1H  CHECKSUM
```

```c
uint16_t data0_u16 = rx[4] | ((uint16_t)rx[5] << 8);
uint16_t data1_u16 = rx[6] | ((uint16_t)rx[7] << 8);
int16_t dx = (int16_t)data0_u16;  // 仅 TYPE=0x02 按有符号解释
int16_t dy = (int16_t)data1_u16;
```

串口接收必须支持半包、粘包、噪声和校验失败，不能假定一次 `read()` 就是一帧。

## 3. 红色视觉运行数据

| TYPE | INDEX | DATA0 / DATA1 | 发送时机 | 主控动作 |
|---:|---|---|---|---|
| `0x01` 当前目标 | 当前航点号 | `target_x, target_y` 无符号 | 运行时每 100 ms 重发 | 显示/记录，不直接做 PID |
| `0x02` 视觉误差 | 当前航点号 | `dx, dy` 有符号 | 检测到有效光斑的每个视觉循环 | 更新 PID 输入和看门狗时间 |
| `0x03` 任务完成 | `0xFF` | `0,0` | 原点到达或巡边完成时一次 | 制动并标记完成 |
| `0x04` 光斑丢失 | 当前航点号 | `0,0` | 红光斑无效时 | 立即停止 PID 增量并制动/保持 |

误差定义：

```text
红色系统：DX = 当前目标 X - 红光斑 X
            DY = 当前目标 Y - 红光斑 Y

```

图像坐标向右、向下为正。电机正反由 STM32 的轴方向系数处理；如果越调越远，翻转对应轴方向，不要修改视觉协议符号。

原点坐标并没有新增特殊数据格式。例如检测到原点 `(318,242)`，复位运行时发送：

```text
TYPE=01, INDEX=0, DATA0=318, DATA1=242   // 每100ms重发
TYPE=02, INDEX=0, DATA0=DX,  DATA1=DY    // 每个有效视觉循环
```

连续 3 帧进入 8 px 半径后，视觉发送 `TYPE=03`。任一帧丢点会把连续计数清零。

原点复位会在完整 `640×480` 画面搜索红光斑，因此可以从屏幕任意位置闭环返回；这一阶段 FPS 可能低于巡边阶段。屏幕/A4 巡边仍使用矩形扩展 ROI。

## 4. 红色主控的模式与运行命令

下列控制帧在红色链路上是双向的：主控可以发送它们，摄像头顶部/底部触摸按钮也会把同一帧回报给主控。收到触摸回报时，主控要同步当前模式；尤其收到 `TYPE=0x11` 后必须立即制动，不能只等待 150 ms 通信看门狗。

| TYPE | INDEX | 用途 |
|---:|---:|---|
| `0x10` | `0` | 开始，或暂停后继续 |
| `0x11` | `0` | 暂停；主控必须同时立即制动 |
| `0x12` | `0` | 重新识别当前模式 |
| `0x13` | `0` | 选择原点复位 |
| `0x13` | `1` | 选择屏幕正方形巡边 |
| `0x13` | `2` | 选择 A4 巡边 |

推荐操作顺序：

```text
1. 主控先制动
2. 发 TYPE=13 选择模式；也可以在摄像头顶部直接点选
3. 发 TYPE=10 开始；若仍在 ACQUIRE，视觉会在锁定后自动运行
4. 调试时观察绿色锁定框；如果需要人工确认，也可等 WAIT_START 后再发 TYPE=10
5. 用 TYPE=02 持续闭环
6. 收到 TYPE=03 后制动并结束本任务
```

`TYPE=0x12` 只重新识别当前模式；它不会把模式强制切回 A4。命令没有 ACK，必要时可以短时间重复发送同一命令。

## 5. 主控控制循环

```c
// UART 收帧
switch (type) {
case 0x01:
    current_index = index;
    target_x = data0_u16;
    target_y = data1_u16;
    break;
case 0x02:
    error_x = (int16_t)data0_u16;
    error_y = (int16_t)data1_u16;
    last_vision_ms = now_ms;
    vision_valid = true;
    break;
case 0x03:
case 0x04:
    vision_valid = false;
    pid_clear();
    motor_brake();
    break;
}

// 建议 100~200 Hz 定时控制
if (!vision_valid || now_ms - last_vision_ms > 150) {
    pid_clear();
    motor_brake();
} else {
    motor_x(pid_x(error_x) * X_DIRECTION);
    motor_y(pid_y(error_y) * Y_DIRECTION);
}
```

不要在收到 `TYPE=01` 后就按固定时间切换下一点。当前点没有到达时，视觉会重复发同一目标；到达判定和航点推进由 MaixCAM 完成。

## 6. 二十个航点的含义

每条边“五等分”会产生六个几何位置（含两端），但相邻边共享角点：

```text
4 个角 + 4 条边 × 4 个内部点 = 20 个不重复航点
```

代码每边发送 `1/5、2/5、3/5、4/5、5/5`，不重复发送本边起点，因此四边合计正好 20 点并顺时针闭合。

## 7. 绿色主控是另一条独立链路

绿色视觉独立检测红、绿光斑，不能读取红色系统的坐标或状态。绿色主控只需要处理：

| TYPE | 含义 | 主控动作 |
|---:|---|---|
| `0x02` | `DX=red_x-green_x`、`DY=red_y-green_y` | 双轴 PID |
| `0x04` | `INDEX=1/2/3` 分别表示红/绿/两者丢失 | 立即制动并关闭提示 |
| `0x05` | `DATA0=0/1` 未追踪成功/追踪成功 | 驱动连续声光提示 |

绿色主控只发送 `TYPE=0x10` 开始或继续、`TYPE=0x11` 暂停。它不处理红色航点，也不发送红色模式选择。

两套系统都要有暂停功能。比赛中的“同时按暂停键”应通过一个人工动作分别接入两套独立主控（例如双联按键/两路独立输入），不能让红色视觉给绿色视觉发送暂停消息，否则违反相互独立要求。

绿色视觉上电等待本系统主控的 `TYPE=0x10` 开始，也支持 `TYPE=0x11` 暂停和再次 `TYPE=0x10` 继续。主控收到/产生暂停操作时仍应先直接制动，不要把安全停车依赖于串口往返。

## 8. 已完成与仍需联调

| 赛题能力 | 当前视觉代码 | 仍需完成 |
|---|---|---|
| 红光回屏幕原点 | 已实现方框中心检测、单航点闭环 | 上板标定 8 px 与 2 cm 的关系 |
| 屏幕正方形顺时针一周 | 已实现 20 航点 | 验证 30 s、2 cm 指标 |
| A4 固定/旋转巡边 | 已实现旋转矩形、中心线优先和 20 航点 | 现场阈值、曝光、30 s、2 cm |
| 绿光追红光 | 已实现独立红绿检测、误差和成功状态 | PID、2 s/3 cm 标定 |
| 声光提示 | 视觉已提供 `TYPE=05` | 绿色 STM32 驱动蜂鸣器/LED |
| 暂停 | 红色触摸+UART、绿色 UART 已处理 | 两套主控物理按键和立即制动 |
| 通信故障安全 | 提供丢失帧 | STM32 必须实现 150 ms 看门狗 |

屏幕任务用 `SCREEN_EXPECTED_AREA_RATIO` 区分内部 0.5 m 正方形和可能被检测到的更大白屏外框，初值为 `0.52`。它表示正方形面积占 `640×480` 画面的比例，不是物理面积比例；相机位置改变后必须重新标定。屏幕上绿色锁定框必须落在铅笔正方形上，不能落在白屏外边界。

自动测试通过只证明协议、几何和状态逻辑，不等于整机已经满足厘米与秒级指标。联调时至少记录：误差极性、丢点制动、通信超时、实际 FPS、矩形锁定时间、20 点完成时间、2 cm/3 cm 像素换算和声光状态。
